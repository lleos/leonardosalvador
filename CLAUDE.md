## What this is

A static HTML portfolio website for Leonardo Salvador (Product Designer). The site was exported from Adobe Portfolio — there is no build system, no package manager, and no framework.

## Serving locally

```bash
# From the leonardosalvador.com/ subdirectory (uses relative paths):
cd leonardosalvador.com && python3 -m http.server 8080

# Or from the root (index.html uses absolute URLs pointing to leonardosalvador.com):
python3 -m http.server 8080
```

## Repository layout

```
index.html                  # Root entry — uses absolute URLs (for live deployment)
leonardosalvador.com/       # Local-serving version — uses relative paths
  work.html                 # Portfolio gallery (main page)
  about.html                # About page
  *.html                    # Case study pages (nubank, ebanx, mettzer, etc.)
  dist/css/main.css         # Compiled site CSS (from Adobe Portfolio)
  dist/js/                  # Compiled site JS
  site/                     # Translations and other site resources
cdn.myportfolio.com/        # Locally cached CDN assets (images, page-specific CSS)
```

## Key architectural notes

- **Two versions of the same site:** `index.html` at root references absolute URLs (`https://leonardosalvador.com/dist/css/...`) for live deployment. Pages inside `leonardosalvador.com/` use relative paths and work for local browsing.
- **Assets are mirrored locally:** The `cdn.myportfolio.com/` directory contains locally-cached copies of CDN images and per-page CSS that the HTML files reference via relative `../cdn.myportfolio.com/...` paths. This makes the site browsable offline.
- **CSS/JS are compiled Adobe Portfolio output** — `dist/css/main.css` and `dist/js/` are not hand-authored source files; they were generated by the platform.
- **Typography:** Adobe Typekit loaded async via a `<script>` tag in every page `<head>`.
- **No JavaScript interactivity** beyond what Adobe Portfolio bundles — page transitions, responsive nav toggling, and image loading use the platform's built-in JS.

## Editing content

All content is in the HTML files. Each case study page is self-contained. Common elements (nav, social links) are duplicated across every page — changes to navigation or social links must be applied to each HTML file individually.
